# 0-cold-start-interview

工作流 0：冷启动访谈

触发条件：用户首次使用本 skill，或本 skill 配置文件 `.user-config.md` 不存在 / 显式要求重新配置。

目的：建立用户的能力槽配置档案与法律解释知识库注册表，让本 skill 后续工作时知道：
- 用户已配置哪些工具（法律数据库检索、文档解析、全网搜索、学术论文检索、Word 输出）；
- 用户是否拥有权威评注 / 解释类知识库 skill，可作为法律解释的第一优先级数据源；
- 用户偏好的报告输出格式。

## 一、访谈结构

整个访谈分为六段，每段独立的 AskUserQuestion 调用，绝不一次性追问 4+ 项：

### 第一段：法律解释知识库（核心模块）

先向用户展示价值说明，然后询问。展示文本（原文一字不差）：

> **关于法律解释知识库**
>
> 鉴定式案例分析的灵魂在"涵摄"，而涵摄的精度取决于对构成要件含义的把握。在分析中遇到"显失公平""重大误解""合理期限""善意"等需要价值填充的概念时，民法典相关条文评注或最高人民法院对于民法典及相关司法解释的理解与适用类著作能为要件解释提供专业、丰富、可引用的依据，让分析有据可依、显著提升论证的说服力与学术品位；它们在报告中形成完整脚注，是合格鉴定式报告的必要支撑。
>
> 如果您已自制此类知识库 skill，本插件会按需调用；如尚未制作，本插件将按方法论工具箱完成解释 + 联网检索学术与实务文章作为论据补充。
>
> 您可将其 skill 名称在下一步注册给本插件，本插件将在法律解释优先级最高一档调用。

随后用 AskUserQuestion 询问：

问题："是否已自制可供调用的法律解释 / 评注类知识库 skill？"
选项 A："已有，希望注册到本插件"
选项 B："暂无，由本插件按方法论工具箱完成解释 + 联网检索学术与实务文章作为论据补充"
选项 C："暂时跳过，由我后续自行决定"

如选 A，再追问 skill 名称（多个用逗号分隔）和每个 skill 的覆盖领域简述，写入 `.user-config.md`。

### 第二段：能力槽配置确认

> **重要说明**：本插件依据校验铁律——法条号/条文原文/现行有效性必须经法律检索工具实时校验，模型记忆不得作为引用依据。未配置法律数据库检索能力槽时允许降级（所有法条带软标注 `[未经校验，请人工核对]`）。

用 AskUserQuestion（一次性确认所有能力槽）：

问题："请确认您当前的工具可用情况（多选）："
选项：
- "S1 法律数据库检索已配置（如各类法律数据库 MCP、法律检索 API 等）"
- "S2 文档解析工具已配置（如 PDF/Word/图片转 Markdown 工具）"
- "S3 全网搜索工具已配置（如联网搜索 MCP、内置 WebSearch 等）"
- "S4 Word 输出工具已配置（如 Markdown 转 Word 工具 / skill）"

未勾选"S1 法律数据库检索"时，向用户做一次显式提醒："本次会话所有法条引用将带 `[未经校验，请人工核对]` 标记，请您在终稿交付时人工逐条核验。"该提醒同步写入 `.user-config.md`。

如已勾选 S1，追问："您配置的法律数据库检索工具的名称和调用方式是什么？（如 MCP 服务器名 / 工具名称等）"

其他未勾选工具在使用时优雅降级。

### 第三段：可选增强能力槽

用 AskUserQuestion 询问可选能力槽：

问题："以下可选能力槽，您是否已配置？（多选）"
选项：
- "S5 学术论文检索工具已配置（如学术论文检索 MCP 等）"
- "无额外工具需要注册"

### 第四段：报告输出格式偏好

问题："输出报告的格式偏好？"
选项 A："学术格式（A4、9 级标题体系、宋体、脚注完整、含案情/设问/分析/结论的完整结构）（推荐）"
选项 B："Markdown 原稿（不转 Word，便于二次编辑）"

### 第五段：写作语言

问题："分析报告的语言？"
选项 A："中文（简体）"（默认）
选项 B："中文 + 英文术语对照"

### 第六段：保存配置

将访谈结果写入技能安装目录下的 `.user-config.md`（即与本文件同级的 `.user-config.md`），模板：

```markdown
# 用户配置（gutachten-civil-case）

最后更新：YYYY-MM-DD

## 法律解释知识库注册

skill1：
skill2：
（无 → 留空 / 注明"未注册"）

## 能力槽配置

### S1 法律数据库检索（必须 · 校验铁律）
状态：已配置 / 未配置（降级模式：所有法条带 [未经校验]）
工具名称：[用户已配置的法律检索工具名称]
调用方式：[MCP 服务器名 / 工具名 / API 端点等]

### S2 文档解析
状态：已配置 / 未配置
工具名称：[用户配置的文档解析工具]

### S3 全网搜索
状态：已配置 / 未配置
工具名称：[用户配置的全网搜索工具]

### S4 Word 输出
状态：已配置 / 未配置
工具名称：[用户配置的 Word 输出工具]

### S5 学术论文检索
状态：已配置 / 未配置
工具名称：[用户配置的学术论文检索工具]

## 报告输出格式偏好

学术格式 / Markdown 原稿

## 写作语言

中文 / 中文+英文术语对照
```

## 二、对未注册解释类 KB 的用户

后续在写作流程中遇到需要法律解释的要件时：
1. 跳过"调用用户 KB"层；
2. 直接进入"调用案例 + 联网检索"层；
3. 仍无依据时进入"自助式解释"层（真实来源原则：自助式方法论解释属正文论证密度的内生组成，不加任何脚注；严禁 [模型推理] [此观点尚需查证] 等占位脚注）。

具体路径见 workflow/3-drafting.md 中"法律解释方法的运用（贯穿增强 + 素材分级）"部分。

## 三、何时再次触发本访谈

- 用户显式要求"重新配置 gutachten-civil-case"；
- `.user-config.md` 文件被删除；
- 用户报告新增 / 卸载了相关工具（手动触发）。

平日不重复访谈，每次会话直接读取 `.user-config.md`。
