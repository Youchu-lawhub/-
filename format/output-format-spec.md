## 三、字体与 9 级标题字号矩阵

| 层级 | Markdown 标记 | Word 样式名 | 字体 | 字号 | 加粗 | 用途 |
| --- | --- | --- | --- | --- | --- | --- |
| 部分 | `# 第X部分` | Heading 1 | 黑体 | 小三（15pt） | 是 | 多对当事人时的分组层 |
| 一级 | `## （一）大纲` / `## （二）正文` | Heading 2 | 黑体 | 小三（15pt） | 是 | 外壳层（大纲 / 正文） |
| 二级 | `### 1. 请求权已产生` | Heading 3 | 黑体 | 四号（14pt） | 是 | 三层四步主层级 |
| 三级 | `#### (1) 合同成立` | Heading 4 | 黑体 | 小四（12pt） | 是 | 要件展开层 |
| 四级 | `##### ① 要约` | Heading 5 | 宋体 | 小四（12pt） | 否 | 子要件层 |
| 五级 | `###### A. 要约的构成` | Heading 6 | 宋体 | 小四（12pt） | 否 | 细分论证层 |
| 六级 | `**a. 具体要素**` | 加粗段落 | 宋体 | 小四（12pt） | 是 | 段头加粗 |
| 七级 | `**(a) 细节**` | 加粗段落 | 宋体 | 小四（12pt） | 是 | 段头加粗 |
| 八级 | `**(aa) 更细节**` | 加粗段落 | 宋体 | 小四（12pt） | 是 | 段头加粗 |
| 九级 | `**(aaa) 最细节**` | 加粗段落 | 宋体 | 小四（12pt） | 是 | 段头加粗 |
| 正文 | 无标记 | Normal | 宋体 | 小四（12pt） | 否 | 正文段落 |
| 引用块 | `> ` | Quote | 楷体_GB2312 | 小四（12pt） | 否 | 法条原文引用 |

## 四、Markdown → Word 层级映射

转换工具在将 Markdown 转为 Word 时，应按以下映射关系生成样式：

| Markdown 元素 | Word 样式 | 说明 |
| --- | --- | --- |
| `# H1` | Heading 1 | 部分层（条件化） |
| `## H2` | Heading 2 | 外壳层（大纲 / 正文） |
| `### H3` | Heading 3 | 三层四步主层级 |
| `#### H4` | Heading 4 | 要件展开层 |
| `##### H5` | Heading 5 | 子要件层 |
| `###### H6` | Heading 6 | 细分论证层 |
| `**加粗段落**` | Bold + Normal | 六~九级段头加粗 |
| `> 引用块` | Quote（自定义样式） | 楷体_GB2312 + 左侧灰线 |
| 正文段落 | Normal | 宋体小四 |
| `[^1]` 脚注 | Footnote | Word 原生脚注 |
| `---` 分隔符 | 分页符 | 部分间分页 |
| `  ` 全角空格 | 首行缩进 2 字符 | 段首缩进 |

**特殊处理：**

- 引用块样式须包含：左侧 2pt 灰色竖线 + 左缩进 1cm + 楷体_GB2312 字体；
- 脚注须转为 Word 原生脚注（非尾注），编号自动连续；
- 分隔符 `---` 在部分间使用时转为分页符，在部分内使用时转为段落间距。

## 六、引用块（法条原文）

引用块用于完整展示被引用法条的原文，须遵循以下视觉样式：

| 属性 | 数值 | 说明 |
| --- | --- | --- |
| 字体 | 楷体_GB2312 | 区别于正文宋体 |
| 字号 | 小四（12pt） | 与正文一致 |
| 左侧竖线 | 2pt 灰色（#CCCCCC） | 视觉分隔 |
| 左缩进 | 1 cm | 整体缩进 |
| 背景色 | 无 | 保持简洁 |
| 段前 / 段后 | 6pt | 与正文段落间距 |

**Markdown 写法：**

```markdown
> 《民法典》第 509 条第 1 款："当事人应当按照约定全面履行自己的义务。"
```

**Word 渲染效果：**

由 `[SLOT-WORD-OUTPUT]` 自动渲染为带左侧灰线的引用块样式。若转换工具不支持自定义样式，则手动调整：选中引用段落 → 设置楷体_GB2312 → 添加左侧竖线边框。

## 七、脚注（真实来源原则，可选不必填）

脚注仅用于实际调用并取得对应段落 / 页码 / 案号 / URL 的外部素材。自助式解释不加脚注，在正文展开方法步骤。

| 属性 | 数值 | 说明 |
| --- | --- | --- |
| 位置 | 脚注（非尾注） | 页面底部显示 |
| 编号 | 自动连续 | Word 自动编号 |
| 字号 | 小五（9pt） | Word 默认脚注字号 |
| 格式 | 统一学术引用格式 | 见 methodology/gutachten-writing-rules.md |

**来源标识符四种且仅四种：**

| 标识符 | 含义 | 使用场景 |
| --- | --- | --- |
| `[评注]` | 法律评注类文献 | 用户 KB 中的评注 / 释义内容 |
| `[案例]` | 司法案例 | 通过 `[SLOT-LEGAL-DB]` 检索的案例 |
| `[联网]` | 网络资源 | 通过 `[SLOT-WEB-SEARCH]` 检索的内容 |
| `[KB: <skill-name>]` | 用户私有知识库 | 从用户注册的 KB 中获取的内容 |

**禁止事项：**

- 禁止 `[模型推理]` / `[此观点尚需查证]` 等占位脚注；
- 禁止"参前注 X"等前向引用；
- 禁止未实际调用 KB 即标注 `[KB:]`。

## 九、Markdown 草稿格式

写作阶段输出的 Markdown 草稿须遵循以下格式：

| 规则 | 说明 |
| --- | --- |
| 段首缩进 | 使用 `  `（U+3000 全角空格 ×2），不用半角空格 |
| 标题层级 | 严格遵循 9 级标题矩阵，不跳级 |
| 引用块 | 使用 `> ` 标记，每段引用单独成块 |
| 脚注 | 使用 `[^n]` 语法，脚注内容在文末集中定义 |
| 分隔符 | 部分间使用 `---`，部分内不使用 |
| 当事人称谓 | 全文统一（如"原告甲公司""被告乙"） |
| 法条引用 | 《法规名》第 XX 条（书名号 + 空格） |
| 校验尾标 | 见 canonical-report-template.md §五 校验尾标三选一规则 |

**草稿示例片段：**

```markdown
# 第一部分

## （二）正文

### 1. 请求权已产生

#### (1) 合同成立

  本案涉及甲与乙之间是否成立了有效的买卖合同。须审查双方是否就标的物、数量、价格等必要之点达成合意。

##### ① 要约

  乙于 2025 年 3 月 1 日向甲发出书面报价，载明……

> 《民法典》第 472 条："要约是希望与他人订立合同的意思表示，该意思表示应当符合下列条件：（一）内容具体确定；（二）表明经受要约人承诺，要约人即受该意思表示约束。"

  乙的报价内容具体确定，且表明一经甲接受即受约束，构成要约。[^1]

[^1]: [案例] 最高人民法院（2020）最高法民终 XXX 号民事判决书。
```

## 十、转换工具优先级

将 Markdown 草稿转为 Word（.docx）时，按以下优先级选择转换方案：

| 优先级 | 工具 | 说明 |
| --- | --- | --- |
| 1 | 用户通过 `[SLOT-WORD-OUTPUT]` 注册的 Word 转换工具（推荐） | 已配置样式映射，可直接输出符合规范的 .docx |
| 2 | 通用 Markdown → Word 转换方案（需手动处理样式映射） | 如 pandoc、其他转换工具，需手动调整字体 / 行距 / 引用块样式 |
| 3 | 兜底：输出 Markdown，提示用户用 Pandoc 自行转换 | 提供转换命令参考，用户自行完成 |

**Pandoc 转换命令参考（兜底方案）：**

```bash
pandoc input.md -o output.docx \
  --reference-doc=template.docx \
  --dpi=300
```

其中 `template.docx` 须预先配置好样式（字体 / 字号 / 行距 / 引用块样式）。
