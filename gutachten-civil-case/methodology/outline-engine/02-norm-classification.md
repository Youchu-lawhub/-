# 02-norm-classification

**Step 2：规范属性识别（Norm Classification）**

对每一个候选条文精确判定其规范属性，这是构建检视程式的前提。

## 一、为什么规范属性识别如此重要

一个法条放在检视程式的不同位置，意味着完全不同的法律后果和举证责任分配：

- **作为主要规范**：它是请求权基础，原告须证明其全部构成要件
- **作为辅助规范**：它辅助主要规范的某个要件，原告仍负举证责任
- **作为防御规范**：它阻却/消灭/阻止请求权，被告负举证责任

错误归类 = 检视程式结构错误 = 整个分析的逻辑基础坍塌。

## 二、18类规范判定规则

### 2.1 主要规范（请求权基础）

识别特征：

- 条文结构同时包含完整的构成要件和明确的法律效果
- 法律效果为"可以请求""应当""有权请求""负有……义务"
- 独立地赋予一方当事人向另一方当事人主张特定给付的权利

判定公式：如果能回答"谁(主体)得基于此条文向谁(相对人)主张什么(法律效果)" → 主要规范

典型标志：

- "……的，应当承担赔偿责任"（第500条）
- "……有权请求……返还"（第235条）
- "……应当承担继续履行、采取补救措施或者赔偿损失等违约责任"（第577条）
- "……造成他人损害的，应当承担侵权责任"（第1165条第1款）

### 2.2 不完全主要规范（归属规范）

识别特征：

- 确定请求权或给付义务的主体归属
- 须与其他规范结合才构成完整请求权基础
- 不独立赋予请求权

典型举例：

- 第1191条（用人单位对外承担责任）——确定谁是侵权责任主体
- 第104条第1句（法人的民事权利能力）
- 第1198条（安全保障义务人）——确定谁负有义务

### 2.3 辅助规范（典型辅助规范）

识别特征：

- 为主要规范的某个构成要件提供定义、解释、补充说明
- 不独立产生法律效果
- 原告援引主要规范时"顺带"援引辅助规范以充实要件内涵

判定公式：此条文是否仅在解释/补充某个主要规范的某个要件时才有意义？ → 是则为辅助规范

典型标志：

- "……是指……"（定义条款）
- "……包括……"（列举条款）
- "……适用……的规定"（参引条款）
- "以下情形之一的，视为……"（拟制条款）

### 2.4 外部规范

识别特征：置入民法中的公法规范，不产生民事请求权但影响民事行为效力

典型举例：第212条（不动产物权登记）、第153条第1款中"法律、行政法规的强制性规定"

### 2.5 宣导规范

识别特征：仅具宣示引导意义，无明确构成要件和法律效果

典型举例：第3条（民事权益受保护）、第4条（平等原则）

### 2.6 法律原则

识别特征：无明确适用前提与法律效果，但具有规范补充和价值指引功能

典型举例：第5条（自愿原则）、第6条（公平原则）、第7条（诚信原则）

### 2.7 裁判指引规范

识别特征：指引法官裁量/判断的规范，不直接对应当事人举证

典型举例：第142条（意思表示解释规则）、第510条（合同漏洞填补）

### 2.8 参引规范

- **提示参引规范**：直接指向被参引规范（"依照……规定"）
  - 如第617条 → 指向第582条
- **类推参引规范**：需借助类推操作（"参照适用……"）
  - 如第647条 → "参照适用"承揽合同有关规定

### 2.9 防御规范

#### 2.9.1 抗辩规范（狭义抗辩）

识别特征：

- 引入新的消极要件（新事实），阻却/消灭/阻止请求权
- 被告须对抗辩事实承担举证责任（本证）
- 典型位于"但书"或独立条文

判定公式：被告是否须主张并证明一个独立于原告请求权构成要件之外的新事实？ → 是则为抗辩

典型标志：

- "但是，……的除外"（但书型）
- "……的，不承担责任"（免责型）
- "……可以请求……撤销"（形成抗辩）
- "……有权拒绝……"（停止性抗辩）

#### 2.9.2 反抗辩规范

抗辩规范的抗辩——原告排除被告抗辩的规范

如第194条第1款（时效中断事由）对抗第192条第1款（时效抗辩）

#### 2.9.3 再抗辩规范

反抗辩规范的抗辩——被告再次排除原告反抗辩的规范

如第194条第2款对抗第194条第1款

#### 2.9.4 否认规范

识别特征：

- 法效体现为对某一构成要件事实的否认
- 不改变被否认方的举证责任（反证即可）
- 与狭义抗辩的关键区别：否认不引入新事实，只否认旧事实

典型举例：第1224条第1款第2、3项（不属于医疗损害中的"诊疗活动"）

#### 2.9.5 抗辩排除规范 / 否认排除规范

说明某类事由不构成抗辩/否认事由

如第215条第2分句（未办登记不影响合同效力 → 排除"未登记"作为合同无效抗辩）

### 2.10 跨类规范

兼具两类属性：

- **主要/辅助**：如第366条（居住权），在确认之诉中为主要规范，在其他场景为辅助规范
- **主要/抗辩**：如第317条第1款（拾得人费用请求权 vs 返还原物的留置抗辩）
- **辅助/抗辩**：兼具辅助和防御功能

## 三、判定的操作流程

```
Step 1: 条文是否独立赋予一方请求权？
  → 是 → 主要规范 / 不完全主要规范
  → 否 → 进入 Step 2

Step 2: 条文是否服务于某主要规范的构成要件？
  → 是 → 辅助规范（含外部/宣导/原则/裁判指引/参引等子类）
  → 否 → 进入 Step 3

Step 3: 条文是否阻却/消灭/阻止某请求权？
  → 是 → 防御规范（抗辩/反抗辩/再抗辩/否认/排除）
  → 否 → 检查是否为跨类规范或程序性规范
```

## 四、区分否认与抗辩的核心技术

这是规范属性识别中最容易出错的环节：

实操判别法：

问自己：被告的防御是在说"你说的不是真的"（否认）还是"即使你说的是真的，但还有另一个事实导致你不能赢"（抗辩）？

| 维度 | 否认 | 狭义抗辩 |
| --- | --- | --- |
| 事实关系 | 否定对方主张事实的真实性 | 承认对方事实，但主张另有新事实 |
| 举证责任 | 不改变（被否认方仍负举证责任） | 转移（抗辩方对抗辩事实负举证责任） |
| 证明标准 | 反证（动摇法官心证即可） | 本证（使法官形成确信） |
| 在检视程式中的位置 | 构成要件项下（作为"不成就"的论证） | 独立抗辩项下（新层级） |

## 五、条文结构与规范属性的对应关系

### 5.1 条文内部结构解剖

一个完整的法律条文可能包含多个规范：

```
第XXX条 [条旨]
  第1款 [主文]                ← 可能是主要规范
    但是，……的除外。          ← 但书部分可能是防御规范
  第2款 [补充说明]            ← 可能是辅助规范
  第3款 [例外情形]            ← 可能是抗辩排除规范
```

### 5.2 "但书"的规范属性判定

并非所有但书都是抗辩：

- **限缩型但书**：限制主文的适用范围 → 辅助规范（如第580条第1款但书：不能强制履行的情形）
- **例外型但书**：对主文效果的排除 → 抗辩规范（如第1165条中的"能够证明自己没有过错的，不承担侵权责任"）
- **确认型但书**：确认某情形不受主文约束 → 否认规范

## 六、输出格式

对每个候选条文输出：

```
条文号：第XXX条第X款
条旨：[一句话说明条文功能]
规范属性：[主要 / 辅助 / 防御-抗辩 / 防御-否认 / ...]
在检视程式中的位置：[请求权已产生-成立要件 / 未消灭-消灭抗辩 / 可行使-行使抗辩 / ...]
举证责任：[原告/被告]
关联条文：[与本条配合使用的辅助/防御规范清单]
```
