# 05-outline-assembly

**Step 5：大纲成型（Outline Assembly）**

> 将前四步的产出组装为完整的解题大纲——这是提交给用户确认（W2闸二）的核心交付物。

## 一、大纲的本质

大纲 = 检视程式 + 案件事实映射 + 争点预判 + 写作指引

它回答的问题是："这个案件的鉴定式分析报告，每一节写什么、按什么顺序写、哪里是重点。"

## 二、大纲的组装规则

### 2.1 外壳层：报告整体结构

```
目录
第X部分：[A → B 的请求权检视]          ← 部分层（≥2对当事人才加）
（一）大纲
  [本部分的检视程式结构化展示]
（二）正文
  一、[诉请1名称]——基于[请求权基础条文]的请求权
    （一）请求权已产生
      1. [第一个构成要件]
        (1) [子要件a]
        (2) [子要件b]
      2. [第二个构成要件]
      ...
    （二）请求权未消灭
      1. [消灭事由1]
      ...
    （三）请求权可行使
      1. [行使障碍1]
      ...
    （四）小结
  二、[诉请2名称]——基于[另一请求权基础]的请求权
  ...
结论
```

### 2.2 大纲项的信息密度

每个大纲项须包含：

```
[要件名称]：第XXX条第X款
├── 案件事实对应：[事实中哪些内容涉及此要件]
├── 初步判断：[成就 / 不成就 / 争议（核心争点）]
├── 论证深度：[明显→简略 / 争议→中等 / 核心争点→深入]
└── 需法律解释：[是→标注具体问题 / 否]
```

### 2.3 争点标记系统

| 标记 | 含义 | 论证深度 |
| --- | --- | --- |
| ◎ | 核心争点（案件成败关键） | 深入论证（含法律解释四步） |
| ○ | 次要争议（存在讨论空间） | 中等论证 |
| · | 明显成就/不成就 | 简略涵摄（一两句话） |

## 三、组装流程

**Step 5.1：排序**

按外在结构优先级排列各法律关系对和各诉请：

1. 当事人：原告 → 被告 → 第三人
2. 请求权类型：合同 → 类似合同 → 无因管理 → 物权 → 侵权 → 不当得利
3. 聚合请求权分别独立列章；竞合请求权分别列章但标注择一

**Step 5.2：填充**

对每个请求权基础：

1. 将 Step 3 产出的检视程式作为骨架
2. 将 Step 4 产出的要件拆解填入每个节点
3. 根据案件事实，为每个要件标注事实对应和初步判断
4. 标记争点等级

**Step 5.3：裁剪**

根据案件复杂度和比例原则裁剪大纲：

- **明显不成就的请求权**：保留检视但标注"本请求权不成立，理由如下……"（简述即可）
- **明显成就且无争议的要件**：折叠为一行（"要件①-③明显成就"）
- **核心争点**：展开至最大详细度

**Step 5.4：交叉引用**

处理多个请求权之间的交叉引用：

- "合同有效成立"在违约请求权和合同解除请求权中重复出现 → 标注"参见上文一、（一）1."
- 侵权请求权中"损害"与违约请求权中"损害"部分重合 → 标注关联

## 四、大纲的输出模板

### 4.1 标准大纲格式（提交用户确认）

```markdown
# 解题大纲

## 案件基本信息
- 当事人：A（原告）、B（被告）[、C（第三人）]
- 法律关系对：A→B [、B→A、A→C]
- 识别的诉请：[编号列出]

## 请求权检视清单

### A → B

#### 诉请一：[具体诉请内容]
请求权基础：第XXX条第X款（[条旨]）

| 序号 | 要件 | 法条 | 事实对应 | 判断 | 争点 |
| --- | --- | --- | --- | --- | --- |
| 1 | [要件名] | §XXX | [事实摘要] | 成就/争议 | ◎/○/· |
| 2 | ... | ... | ... | ... | ... |

（一）请求权已产生

产生抗辩（被告方）：

| 序号 | 抗辩事由 | 法条 | 事实对应 | 判断 |
| --- | --- | --- | --- | --- |
| 1 | [抗辩名] | §XXX | [事实摘要] | 成立/争议/不成立 |

（二）请求权未消灭
[同上格式]

（三）请求权可行使
[同上格式]

小结：本请求权[成立/不成立/取决于争点X的解决]

#### 诉请二：...

## 写作策略建议
- 核心争点集中在：[列出]
- 建议论证重点：[列出]
- 预计字数分配：诉请一（X%）/ 诉请二（Y%）/ 结论（Z%）
- 法律解释重点：[要件X的含义界定]

## 聚合/竞合关系
- 聚合：诉请1与诉请2可同时主张
- 竞合：诉请1与诉请3基于同一事实，原告择一行使
```

### 4.2 精简大纲格式（内部工作用）

```
A→B / 诉请1 / §577（违约赔偿）
产生：合同有效成立(·) → 义务违反(○) → 归责性(·) → 损害(◎) → 因果关系(○)
消灭：清偿(·否) / 减损义务(○)
行使：同时履行(·否) / 时效(·否)
结论：成立，取决于损害范围
```

## 五、从大纲到写作的衔接

大纲确认（闸二通过）后，进入 W3 写作阶段时：

1. **大纲作为骨架**：W3 严格按大纲的章节结构和顺序展开
2. **争点标记指导论证深度**：◎=深入论证，○=中等论证，·=简略涵摄
3. **法律解释标记触发方法论工具箱**：标注"需法律解释"的要件，写作时必须展开文义/体系/历史/目的分析
4. **大纲不可突破但可微调**：写作过程中发现新争点，须报告但不得自行重构大纲

## 六、大纲生成的质量标准

- [ ] 是否覆盖了所有可能的请求权基础（不遗漏）？
- [ ] 是否正确排列了检视顺序（合同→物权→侵权→不当得利）？
- [ ] 每个请求权基础是否完整走完三层？
- [ ] 争点标记是否合理（核心争点不超过2-3个）？
- [ ] 案件事实中的每个法律相关事实是否都在某个要件下有对应？
- [ ] 是否标注了需要法律解释的不确定概念？
- [ ] 聚合/竞合关系是否明确？
- [ ] 写作策略是否清晰？
