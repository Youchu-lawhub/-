# 2b-outline

工作流 2b：解题大纲（闸二）

目的：基于 2a 已独立确认的候选请求权清单，展开为符合鉴定式三阶段（请求权已成立 / 请求权未消灭 / 请求权可行使）的逐要件解题大纲，每要件后括注法规简称 + 条文序号 + 校验状态，提交用户第二闸独立确认。

> 本工作流硬性边界：
>
> 1. 进入本阶段的前提是 2a 闸一已独立拍板——若 AI 自检发现 2a 输出中夹带了大纲，2a 即视为违例，本阶段必须先回退 2a 重做。
> 2. 本阶段产物只能是大纲，不得退回去重排候选清单（候选清单已在 2a 锁定）。
> 3. 大纲中所有条文号必须能在 verification-log.md 中查到 ✓ 或 ⚠ 标记；新增条文须先即时经 `[SLOT-LEGAL-DB]` 校验后追加 log 再写入大纲。

## 一、读入 2a 的产物

| 输入 | 来源 |
| --- | --- |
| 候选请求权清单 | 2a-preselection.md |
| 法律数据库校验台账 | verification-log.md |
| W1 法律关系对 | 1-case-intake.md |

## 二、为每个候选请求权展开三阶段大纲

每条请求权按 methodology/inner-structure.md 三阶段层叠展开：

### 模板（单个请求权）

```markdown
## 一、《民法典》第 X 条 + 第 Y 条 [请求权名称]（候选）

### （一）请求权是否已成立

1. 责任成立 / 合同关系成立 / 物权变动 / ……
   (1) [要件 1]（民法典 X.X [✓]）
   (2) [要件 2]（民法典 Y.Y [✓]）
   (3) [核心争点要件]（民法典 Z [✓]）—— **核心争点**
   (4) 抗辩之一：[要件]（民法典 X [✓]）
   (5) 抗辩之二：[要件]（民法典 X [✓]）

2. 责任范围 / 给付范围
   (1) 损害项目（民法典 X [✓]）
   (2) 责任范围因果关系

3. 请求权主体
   (1) [主体规则]（民法典 X [✓]）

### （二）请求权是否已消灭

- 一般消灭事由（民法典 557.1 [✓]）

### （三）请求权是否可行使

- 诉讼时效（民法典 188 / 192.1 [✓]）
- 同时履行抗辩（民法典 525 [✓]，仅双务合同）
- 不安抗辩（民法典 527 [✓]，仅符合条件）

### （四）小结

（结构层小结：(一) 任一不成立则后续标"无须检视"；本节仅写大纲不下结论）
```

### 关于核心争点的标注

在大纲中显性标记核心争点（用 `—— **核心争点**` 后缀）。这是 W3 写作阶段五要素 ≥500 字密度阀的指引，不能在大纲阶段省略。

### 关于 (一) 任一不成立后的标注

大纲中应预留"若 X 不成立 → 后续标'无须检视'"的备忘，方便 W3 写作时按层级跳过。

## 三、闸二确认（独立拍板）

用 AskUserQuestion：

问题："以下为解题大纲（每要件已括注法规简称 + 条文序号 + 校验状态）。请确认或修正："
选项 A："大纲完整正确，进入 W3 正文写作"
选项 B："需调整三阶段切分 / 要件顺序 / 核心争点标注"
选项 C："需补充或删减某要件下的条文"

可在闸二一并就派生写作选择问询（这些是大纲所需的写作策略决定，与"大纲本身是否准确"分属两类）：

- 多个候选请求权的展开范围（全上 / 仅主请求权 / 部分组合）
- 结论取向（AI 独立论证 / 倾向支持 / 倾向不支持）
- 抗辩条款是否独立展开（如 1173 与有过失）
- 是否调用评注/解释类知识库（如有）

严禁把"派生写作选择"提前到闸一——闸一只问"候选清单本身是否准确"。

## 四、并行写作预备

- 大纲中法律关系对 ≥ 2 时，主流程将每对登记为独立写作单元，并传入 verification-log.md 路径供 W3 复用。
- 单一法律关系对（如丙 → 乙）时不强制并行 subagent，由主线程顺序写。

## 五、与下游衔接

| 下游 | 接收 |
| --- | --- |
| W3 写作 | 大纲中的要件层级 + 括注条文号 + 核心争点标记 |
| W4 核验 | 大纲覆盖度（W3 是否每要件都展开）+ 校验台账（条文真实性） |

## 六、违例自检

如果你正在生成本阶段产物且发生以下任一项 → 立即停止：

1. 闸一未拍板就直接出大纲（相当于合并两闸）
2. 大纲中出现 verification-log.md 中没有的条文号
3. 闸二询问中夹带"候选请求权清单是否调整"——这本应是闸一已锁定项
4. 候选请求权清单与大纲合并在同一文件输出

任一命中均视为方法论违例（与来源伪造同级）。
