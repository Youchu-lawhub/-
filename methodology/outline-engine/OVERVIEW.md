# OVERVIEW

请求权基础解题大纲生成引擎

> 本模块是鉴定式案例分析工作流的思维内核——一套不依赖预构建知识库、能从第一性原理自主完成"确定请求权基础 → 构建检视程式 → 拆解构成要件 → 生成解题大纲"的方法论引擎。

## 一、设计哲学

传统路径：案由 → 在知识库中查找预制检视程式 → 套用。

本引擎路径：事实 → 找法（规范发现）→ 识别规范属性 → 自主构建检视程式 → 要件拆解 → 大纲成型。

区别在于：前者是"查字典"，后者是"会造句"。本引擎让 AI 掌握民法典规范体系的任督二脉——具备从任何法条出发、自主搭建完整三层检视程式的能力。

## 二、五步流水线

```
┌─────────────────────────────────────────────────────────────────┐
│  Step 1: 找法（Norm Discovery）                                    │
│  从案件事实中提取法律关系 → 识别诉请类型 → 锁定候选请求权基础条文     │
├─────────────────────────────────────────────────────────────────┤
│  Step 2: 规范属性识别（Norm Classification）                        │
│  对每个候选条文判定：主要规范 / 辅助规范 / 防御规范 / 跨类规范        │
├─────────────────────────────────────────────────────────────────┤
│  Step 3: 构建检视程式（Schema Construction）                        │
│  匹配骨架库模板 → 案件事实适配 → 挂载辅助/防御规范                    │
├─────────────────────────────────────────────────────────────────┤
│  Step 4: 要件拆解（Element Decomposition）                          │
│  将每层每条文拆解为最小构成要件单元，标注需解释的不确定概念             │
├─────────────────────────────────────────────────────────────────┤
│  Step 5: 大纲成型（Outline Assembly）                               │
│  组装为完整的解题大纲，含三层结构 + 攻防链 + 法条引用 + 争点标记       │
└─────────────────────────────────────────────────────────────────┘
```

## 四、与现有工作流的整合点

本引擎嵌入 gutachten-civil-case 工作流的以下位置：

- W2 闸一（2a-preselection.md）：调用 Step 1-2，输出候选请求权基础清单
- W2 闸二（2b-outline.md）：调用 Step 3-5，输出完整解题大纲
- W3 写作（3-drafting.md）：大纲作为写作骨架

与用户知识库的关系：

- 用户知识库可用时：优先参考知识库的预制检视程式，本引擎作为校验与补充
- 用户知识库不可用时：本引擎独立完成全部检视程式构建（核心价值所在）
- 两种路径产出的检视程式可互相比对验证

## 五、法律数据库工具依赖

本引擎通过 `[SLOT-LEGAL-DB]`（用户注册的法律数据库检索工具）实现法条的实时获取与校验。具体工具由用户在冷启动访谈中配置。

## 六、质量保障

1. 闭环验证：对于任何检视程式，可将其输出与用户已注册的请求权基础知识库（如有）进行比对验证
2. `[SLOT-LEGAL-DB]` 校验：所有法条引用必须经 `[SLOT-LEGAL-DB]` 实时校验现行有效性
3. 方法论自检：每个检视程式须通过"完备性检查清单"（见 03-schema-construction.md §四）
