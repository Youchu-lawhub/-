# 06-general-framework

**通用检视程式骨架库**

> 从《民法典》请求权基础方法实用规范指引中提炼的轻量通用模板，覆盖绝大多数案例场景。
> 本文件不替代法律数据库实时检索，而是提供快速构建检视程式的起点框架。

**凡例**

- 阿拉伯数字指向《民法典》条-款，如"509.1"=第509条第1款
- "句"=句，如"496.2句2"=第496条第2款第2句
- "分句"=分句，如"584分句1"=第584条第一分句
- 项以"(一)(二)(三)"标注，如"497(二)"=第497条第(二)项
- "合通解"=合同编通则司法解释
- "总解"=总则编司法解释
- "担保解"=担保制度司法解释

## 一、合同原给付请求权

**请求权基础选择**

| 情形 | 请求权基础 |
| --- | --- |
| 有名合同+有约定 | 合同条款 + 509.1 |
| 有名合同+无约定 | 分则义务条款（如598买卖、703租赁） |
| 无名合同+有约定 | 合同条款 + 509.1 |
| 无名合同+无约定 | 646参照有偿合同 / 467.1参照最相类似合同 |

**通用检视骨架**

```
请求权已产生
├── 合同成立
│   ├── 要约：472（构成）+ 137/138/139（生效）+ 141（未被撤回）
│   ├── 承诺：479（构成）+ 480（方式）+ 488/489（一致性）
│   └── 特别要件：490（要式）/ 586.1、679、890（要物）/ 496.2句2（格式条款订入）
├── 合同生效
│   ├── 行为能力：144（无行为能力无效）/ 145.1（限制行为能力效力待定）
│   ├── 意思表示无瑕疵：146（虚假行为）/ 147-151（可撤销事由）
│   ├── 内容合法：153.1（违反强制性规定）/ 153.2（违背公序良俗）/ 154（恶意串通）
│   ├── 生效条件：158（附条件）/ 160（附期限）/ 502.2（审批）
│   └── 代理权：162（有权代理）/ 171（无权代理追认）/ 172（表见代理）
├── 履行期限届至
└── 具体给付义务内容

请求权未消灭
├── 557.1（债的消灭事由）
├── 解除：562/563（解除权）+ 565（解除通知）+ 566（解除效果）
└── 撤销权行使：152（除斥期间）

请求权可行使
├── 525（同时履行抗辩）
├── 526（先履行抗辩）
├── 527（不安抗辩）
└── 192.1（诉讼时效抗辩）
```

## 二、违约损害赔偿请求权

**请求权基础选择**

| 情形 | 请求权基础 |
| --- | --- |
| 替代给付赔偿（履行不能/根本违约） | 577后半段第3种情形 |
| 期前拒绝 | 578 |
| 与给付并存的赔偿（迟延/加害给付） | 583 |
| 有名合同特别规定 | 617（买卖瑕疵）/714（租赁）/784（承揽）等 |

**通用检视骨架**

```
请求权已产生
├── 责任成立
│   ├── 合同有效成立（同上）
│   ├── 义务违反
│   │   ├── 义务类型：主给付/从给付/附随义务(509.2)
│   │   ├── 违反形态：不能/迟延/不完全履行/加害给付
│   │   └── 特殊规则：580.1但书（履行不能的判定）
│   └── 可归责性
│       └── 不可抗力抗辩：590（防御规范）
├── 责任范围
│   ├── 损害：584主文（包括实际损失和可得利益）
│   └── 可预见性限制：584但书（防御规范）

请求权未消灭
├── 557.1（通用消灭事由）
├── 591.1（减损义务违反→扣减）
└── 损益相抵（学理）

请求权可行使
├── 525-527（履行抗辩权）
└── 192.1（诉讼时效抗辩）
```

## 五、过错侵权损害赔偿请求权

**请求权基础**

1165.1

### 检视程式一：作为侵权 + 绝对权（标准模板）

```
请求权已产生
├── 责任成立
│   ├── 事实构成 [原告举证]
│   │   ├── 适格权益被侵：109-114、123-126
│   │   ├── 加害行为（作为）
│   │   └── 责任成立因果关系（条件说+相当性）
│   │       ├── 否认：受害人故意 1174
│   │       └── 否认：第三人原因 1175
│   ├── 抗辩：不法性阻却 [被告举证]
│   │   ├── 正当防卫：181.1
│   │   ├── 紧急避险：182.1、182.2
│   │   ├── 紧急救助：184
│   │   ├── 自助行为：1177.1主文
│   │   ├── 自甘冒险：1176.1主文
│   │   └── 受害人同意（学理）
│   ├── 抗辩：欠缺责任能力 [被告举证]
│   ├── 过错（故意/过失）[原告举证]
│   │   └── 暂时丧失意识：1190（醉酒/药物→不免责）
│   └── 抗辩：与有过失（损害发生）：1173 [被告举证]
├── 责任范围
│   ├── 损害 [原告举证]
│   │   ├── 人身损害：1179 + 人身损害赔偿解释6-19
│   │   ├── 财产损害：1184
│   │   ├── 侵害人身权益财产损失：1182
│   │   └── 精神损害：1183（严重精神损害门槛）
│   ├── 责任范围因果关系（相当性+保护目的）
│   ├── 抗辩：与有过失（损害扩大）：1173 [被告举证]
│   └── 分期支付担保：1187.2句2 + 人身损害赔偿解释20-21

请求权未消灭
├── 557.1（债的一般消灭事由）
└── 损益相抵（学理）

请求权可行使
├── 留置抗辩权：类推525
└── 时效抗辩权：192.1（三年，自知道损害+义务人起算：188）
```

### 检视程式二：不作为侵权 + 绝对权

```
请求权已产生
├── 责任成立
│   ├── 适格权益被侵 [原告举证]
│   ├── 作为义务 [原告举证]
│   │   └── 来源：法律规定（1005等）/ 合同 / 先行行为 / 安全保障义务（1198）
│   │       → 作为义务确立后推定：不作为+不法性+过错
│   ├── 否认事由 [被告反证]
│   │   ├── 已尽作为义务 → 否认不作为+不法性+过错
│   │   ├── 无避免可能性 → 否认因果条件性
│   │   ├── 受害人故意 1174 → 否认因果关系
│   │   └── 第三人原因 1175 → 否认因果关系
│   ├── 作为义务保护范围
│   ├── 抗辩：欠缺责任能力
│   └── 抗辩：与有过失（损害发生）：1173
├── 责任范围（同检视程式一）

请求权未消灭（同检视程式一）

请求权可行使（同检视程式一）
```

### 检视程式三：纯粹经济损失（行为不法路径）

```
请求权已产生
├── 责任成立
│   ├── 纯粹经济损失 [原告举证]
│   ├── 加害行为——行为不法性 [原告积极证明]
│   │   ├── 路径①：违反保护性法律 + 保护范围覆盖原告
│   │   └── 路径②：故意悖俗（违反善良风俗+故意）
│   ├── 因果关系
│   │   ├── 否认：受害人故意 1174
│   │   └── 否认：第三人原因 1175
│   ├── 抗辩：不法性阻却
│   ├── 抗辩：欠缺责任能力
│   ├── 可归责性
│   │   ├── 路径①→过错（故意或过失）
│   │   └── 路径②→仅故意
│   └── 抗辩：与有过失 1173
├── 责任范围（同检视程式一）

请求权未消灭（同检视程式一）

请求权可行使（同检视程式一）
```

## 六、物权返还请求权

**请求权基础**

```
请求权已产生
├── 成立要件
│   ├── 请求人为物权人
│   │   ├── 所有权取得：不动产登记/动产交付/继承/善意取得311
│   │   └── 用益物权/担保物权（如适用）
│   └── 相对人为现占有人
├── 成立抗辩
│   └── 占有本权（合同/法定/判决）

请求权未消灭
├── 物已灭失
└── 善意取得311（所有权已转移）

请求权可行使
├── 留置抗辩：类推525
└── 时效排除：196(二)（物权请求权不适用诉讼时效）
```

## 九、无因管理请求权

**请求权基础**

979.1第1分句（费用偿还）/ 979.1第2分句（损害补偿）

**通用检视骨架**

```
请求权已产生
├── 成立要件
│   ├── 管理人支出费用/受到损失
│   ├── 管理他人事务
│   ├── 管理意思（为他人利益）
│   └── 因果关系
├── 成立抗辩
│   ├── 法定或约定管理义务
│   ├── 不符本人真实意思：979.2第1分句
│   └── 抗辩排除：真实意思违法/背俗：979.2但书

请求权未消灭
└── 557.1

请求权可行使
└── 192.1（诉讼时效）
```

## 十、使用说明

1. **本文件定位**：快速起步模板，非完整知识库。遇到以上骨架未覆盖的特殊情形（如特殊侵权、有名合同特殊规则、身份关系请求权等），须通过 `[SLOT-LEGAL-DB]` 检索具体条文自主构建检视程式。
2. **与 `[SLOT-LEGAL-DB]` 的配合**：骨架中每个条文号在正式使用前均须经 `[SLOT-LEGAL-DB]` 校验现行有效性并加载全文。
3. **扩展方法**：任何本文件未覆盖的请求权类型，均可按 Step 1-5 的方法论流水线从零构建，无需依赖预制模板。
4. **本文件来源**：基于民法典及配套司法解释相关规定提炼而成。
