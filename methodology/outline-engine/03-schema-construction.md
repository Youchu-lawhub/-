# 03-schema-construction

**Step 3：检视程式构建（Schema Construction）**

> 以主要规范为核心，按三层结构挂载辅助规范与防御规范，形成完整的检视程式。

## 一、三层结构的本质

检视程式不是"列清单"，而是一棵攻防逻辑树：

```
请求权基础（主要规范）
├── 第一层：请求权已产生
│   ├── 责任成立要件（积极要件，原告举证）
│   │   ├── 要件1 [辅助规范]
│   │   ├── 要件2 [辅助规范]
│   │   └── ...
│   ├── 责任范围（如为损害赔偿请求权）
│   │   ├── 损害 [辅助规范]
│   │   └── 责任范围因果关系 [辅助规范/防御规范]
│   └── 产生抗辩（权利障碍抗辩，被告举证）
│       ├── 无效事由 [防御规范]
│       ├── 撤销事由 [防御规范]
│       └── ...
├── 第二层：请求权未消灭
│   └── 消灭抗辩（权利消灭抗辩，被告举证）
│       ├── 清偿/提存/抵销/混同/免除 [防御规范]
│       ├── 解除 [防御规范]
│       ├── 撤销 [防御规范]
│       └── ...
└── 第三层：请求权可行使
    └── 行使抗辩（停止性抗辩，被告举证）
        ├── 同时履行抗辩 [防御规范]
        ├── 先履行抗辩 [防御规范]
        ├── 不安抗辩 [防御规范]
        └── 诉讼时效抗辩 [防御规范]
```

## 二、构建流程（核心算法）

**Step 3.0：骨架匹配（Template Matching）**

基本逻辑：先从通用检视程式骨架库（06-general-framework.md）中匹配对应模板，再根据具体请求权基础和案件事实做适配调整。

```
已确认的请求权基础（Step 1-2 输出）
→ 在骨架库中定位对应类型模板
→ 加载模板作为初始骨架
→ 进入 Step 3.1 针对本案事实做适配
```

匹配规则：

若骨架库无直接对应模板（如特殊侵权、身份关系请求权等），则跳过本步骤，进入 Step 3.1 从条文出发自主构建。

**Step 3.1：案件事实适配（Case-Specific Adaptation）**

在骨架模板基础上，根据案件事实和已确认的请求权基础做以下调整：

**A. 选择子模板（如适用）**

部分请求权类型在骨架库中有多个子模板（如侵权三套检视程式），须根据案件事实选择：

- **侵权请求权**：判断加害行为方式（作为 / 不作为）+ 权益类型（绝对权 / 纯粹经济利益）→ 选用对应子模板
- **合同请求权**：判断有名合同类型 → 确定对应分则义务条款
- **不当得利**：判断给付型 / 权益侵害型 → 适配不同的"无法律上原因"判定路径

**B. 裁剪与扩展**

- **裁剪**：对与本案事实明显无关的骨架节点，标记"[本案不涉及]"并折叠（不删除，保持结构完整性）
- **扩展**：如案件事实涉及骨架未覆盖的特殊规范（如特别法、地方条例等），在对应层级增加节点

**C. 条文具体化**

骨架模板中的条文号为通用引用；本步骤须结合 `[SLOT-LEGAL-DB]` 实际获取的条文内容，确认每个节点的条文号在本案语境下确实适用。

**Step 3.2：确定主要规范的完整构成要件**

适用场景：骨架库无对应模板时，从主要规范条文出发自主构建。

从主要规范（请求权基础条文）出发，解析其法律效果发生所需的全部前提条件：

操作方法：

1. 加载条文全文（通过 `[SLOT-LEGAL-DB]`）
2. 识别条文中的法律效果部分（"……的，应当/有权/可以……"）
3. 将"……的"之前的全部条件拆解为独立要件
4. 对每个要件判断：是否需要辅助规范来充实含义

**Step 3.3：挂载辅助规范**

对每个构成要件，搜索并挂载解释/补充该要件的辅助规范：

搜索策略：

1. 同条内搜索：第2款/第3款是否解释第1款的要件
2. 同节内搜索：同一节的相邻条文是否提供定义/列举
3. 总则回溯：总则编是否有适用于该要件的一般规定
4. 司法解释搜索：配套司法解释是否细化该要件
5. 法律数据库语义搜索：通过 `[SLOT-LEGAL-DB]` 的语义检索功能检索相关条文

**Step 3.4：挂载防御规范（按三层分布）**

**第一层防御（产生抗辩/权利障碍抗辩）：**

阻却请求权产生的事由：

- 合同请求权：合同不成立/无效/被撤销/未生效
  - 无效事由：第144条/第146条/第153条/第154条
  - 可撤销事由：第147条/第148条/第149条/第150条/第151条
  - 效力待定：第145条/第171条
- 侵权请求权：不法性阻却/无过错
  - 正当防卫：第181条
  - 紧急避险：第182条
  - 自助行为：第1177条
  - 自甘冒险：第1176条
  - 共通：与有过失（第1173条）/受害人故意（第1174条）

**第二层防御（消灭抗辩）：**

使已产生的请求权消灭的事由：

- 通用消灭事由（第557条第1款）：清偿/抵销/提存/混同/免除/法律规定或约定
- 合同解除：第562条/第563条
- 撤销权行使：第152条（除斥期间内行使）
- 减损义务违反：第591条
- 损益相抵：无明文规定，依学理

**第三层防御（行使抗辩/停止性抗辩）：**

使请求权暂时不能行使的事由：

- 同时履行抗辩：第525条
- 先履行抗辩：第526条
- 不安抗辩：第527条
- 诉讼时效抗辩：第192条第1款
- 保证人先诉抗辩：第687条第2款

## 三、结构完备性检查清单

每个检视程式构建完成后，逐项核对：

- [ ] 主要规范是否已正确识别？是否确为请求权基础而非辅助规范？
- [ ] 第一层要件是否穷尽？是否遗漏了隐含要件（如因果关系）？
- [ ] 辅助规范是否正确挂载到对应要件？
- [ ] 产生抗辩是否覆盖了本类型请求权的典型权利障碍事由？
- [ ] 消灭抗辩是否至少包含第557条第1款通用消灭事由？
- [ ] 行使抗辩是否至少包含诉讼时效抗辩（如适用）？
- [ ] 损害赔偿类请求权是否区分了责任成立与责任范围？
- [ ] 每个防御规范是否正确标注了举证责任归属？
- [ ] 是否存在反抗辩/再抗辩需要展开？
- [ ] 特殊规则是否已标注？（如物权请求权不适用时效、侵权精神损害赔偿的特殊门槛）

## 四、检视程式的"可扩展性"

本引擎生成的检视程式具有层级可扩展性：

- **基础版**：仅列出核心要件 + 通用防御规范（适用于快速分析）
- **展开版**：每个要件配齐辅助规范 + 全部潜在防御规范（适用于深度分析）
- **定制版**：基于案件事实裁剪——与本案无关的抗辩可折叠/省略

W2 闸二输出的大纲应为展开版，W3 写作中根据比例原则决定实际论证深度。
