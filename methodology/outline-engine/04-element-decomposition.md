# 04-element-decomposition

**Step 4：要件拆解（Element Decomposition）**

> 将检视程式中的每个构成要件拆解为最小涵摄单元，并标注需法律解释的不确定概念。

## 一、要件拆解的目的

检视程式只是骨架，要件拆解把骨架填成血肉。每个要件最终要变成一个可以直接进行"大前提—小前提—结论"三段论涵摄的最小单元。

## 二、条文结构解析

### 2.1 法律条文的标准结构

```
构成要件（Tatbestand）  →  法律效果（Rechtsfolge）
"如果……"              →  "那么……"
```

解析步骤：

1. 识别法律效果部分（"……应当/可以/有权……"之后的内容）
2. 将法律效果之前的全部条件视为构成要件
3. 将构成要件按"且"（并列）与"或"（选择）拆分为独立单元

### 2.2 连接词的法律意义

| 连接词 | 逻辑含义 | 拆解方式 |
| --- | --- | --- |
| "和""以及""同时" | 并列（AND） | 每个条件独立成为一个要件，全部须满足 |
| "或者""或" | 选择（OR） | 构成选择性要件，满足其一即可 |
| "但是，……除外" | 例外排除 | 但书部分独立为防御规范 |
| "有下列情形之一的" | 列举型选择 | 各项为OR关系 |
| "同时具备下列条件" | 列举型并列 | 各项为AND关系 |

### 2.3 隐含要件的识别

有些要件不在条文文字中直接出现，但法律适用必须检视：

- **因果关系**：损害赔偿请求权中，"行为—损害"之间的因果关系通常不在条文中明言
- **过错**：部分条文不写"有过错的"但解释上需要过错（如第500条缔约过失）
- **违法性/不法性**：侵权法中的不法性通常隐含于"侵害"一词
- **行为能力**：合同请求权中"合同有效成立"隐含行为能力要件

## 三、要件分类

### 3.1 按内容分类

| 要件类型 | 说明 | 举例 |
| --- | --- | --- |
| 主体要件 | 主体资格/身份 | "出卖人""经营者""物权人" |
| 行为要件 | 行为或事件 | "交付""侵害""订立合同" |
| 客体要件 | 行为对象 | "标的物""民事权益""财产利益" |
| 主观要件 | 内心状态 | "故意""过失""善意""恶意""明知" |
| 因果要件 | 因果关联 | "造成""导致""致使" |
| 结果要件 | 损害后果 | "损害""损失""利益受损" |
| 程序要件 | 程序条件 | "经催告""期限届满""通知到达" |

### 3.2 按确定性分类

| 类型 | 特征 | 涵摄难度 | 需要解释？ |
| --- | --- | --- | --- |
| 确定性概念 | 含义清晰、边界明确 | 低 | 通常不需要 |
| 不确定法律概念 | 需价值填充 | 高 | 需要 |
| 一般条款 | 极度抽象 | 最高 | 必须 |

不确定法律概念举例：

- "重大"（重大误解/重大过失/重大事由）
- "合理"（合理期限/合理方式/合理费用）
- "显失公平"
- "善意"
- "公序良俗"
- "根本违约"
- "不可抗力"

## 四、拆解的操作方法

### 4.1 逐句拆解法

以第1165条第1款为例：

> 原文：行为人因过错侵害他人民事权益造成损害的，应当承担侵权责任。

拆解：

- **法律效果**：承担侵权责任
- **构成要件**：
  - ① 行为人 ← 主体要件（辅助规范：第13条/第57条/第102条）
  - ② 过错 ← 主观要件（不确定概念，需解释：故意或过失）
  - ③ 侵害 ← 行为要件（含不法性判断）
  - ④ 他人民事权益 ← 客体要件（辅助规范：第109-114条等）
  - ⑤ 造成损害 ← 结果要件 + 因果要件
- → 进一步拆解：
  - ⑤a. 损害的存在 ← 结果要件
  - ⑤b. 责任成立因果关系 ← 因果要件

### 4.2 层级展开法

当一个要件本身是复合概念时，继续向下拆解：

```
要件"合同有效成立"：
├── 合同成立
│   ├── 要约 [第472条]
│   │   ├── 内容具体确定
│   │   └── 表明受约束的意思
│   ├── 承诺 [第479条]
│   │   ├── 受要约人向要约人发出
│   │   ├── 内容为对要约之同意
│   │   └── 在承诺期限内
│   └── 特别成立要件（如有）
│       ├── 要式要件 [第490条]
│       └── 要物要件（如有）
└── 合同有效（无效力瑕疵）
    ├── 行为能力适格 [第144/145条]
    ├── 意思表示无瑕疵 [第146-151条]
    ├── 内容合法 [第153条]
    └── 生效条件成就 [第158/160条]
```

### 4.3 对不确定概念的处理

当拆解到不确定法律概念时，标注为"需法律解释"：

```
要件"过错"[需法律解释]：
├── 文义边界：故意或过失
├── 故意：明知且追求/放任结果发生
├── 过失：应知而不知/应注意而未注意
│   ├── 一般过失：善良管理人注意义务标准
│   └── 重大过失：连普通人注意义务都未尽到
└── 标注：本案争议在于[具体争点]
```

## 五、不同请求权类型的典型要件拆解模板

### 5.1 合同原给付请求权

- ① 合同有效成立（详见层级展开法）
- ② 履行期限已届至（第509条/第511条）
- ③ 具体给付义务内容（合同条款 + 有名合同分则 + 第510条补充解释）

### 5.2 违约损害赔偿请求权（第577条后半段第3种情形）

**责任成立：**

- ① 合同有效成立
- ② 义务违反（不履行/不完全履行/迟延履行）
- ③ 可归责性（非不可抗力：第590条反面）

**责任范围：**

- ④ 损害（第584条主文：包括实际损失和可得利益）
- ⑤ 因果关系 + 可预见性限制（第584条但书）

### 5.3 过错侵权损害赔偿请求权（第1165条第1款）

**责任成立：**

- ① 民事权益（第109-114条列举）
- ② 加害行为（作为/不作为）
- ③ 责任成立因果关系（条件+相当性）
- ④ 不法性（默认推定，阻却事由为抗辩）
- ⑤ 过错（故意/过失）

**责任范围：**

- ⑥ 损害（第1179-1187条细化）
- ⑦ 责任范围因果关系（相当性+保护目的）

### 5.4 给付不当得利请求权（第985条主文）

**成立要件：**

- ① 相对人取得财产利益（利益概念的界定）
- ② 给付关系（有意识、有目的的增加他人财产）
- ③ 无法律上原因（给付目的落空/自始无原因/嗣后丧失原因）

### 5.5 物权返还请求权（第235条）

**成立要件：**

- ① 请求人为物权人（所有权/用益物权证明）
- ② 相对人为现占有人（直接/间接占有）
- ③ 相对人无占有本权（占有本权为抗辩，被告举证）

## 六、否认与抗辩的结构定位强制判定

本步骤为强制环节：在完成要件拆解后、输出最终检视程式前，必须对每一个防御性规范逐一执行以下判定——

### 6.1 判定流程

对每一个"被告可能援引的条文"：

**Q1**: 该条文是在说"原告主张的某个要件事实不真实"，还是在说"即使原告主张的事实为真，但存在另一个独立事实阻却/消灭/停止请求权"？

- 前者 = 否认 → 挂在被否认要件的下方，标注"否认"
- 后者 = 抗辩 → 独立成项，标注"抗辩"

**Q2**: 如为否认——被告是否需要承担本证义务（使法官形成确信）？

- 否（仅需反证动摇心证）= 确认为否认
- 是 = 重新审视，可能误判为否认，实为抗辩

### 6.2 高频否认/抗辩判定清单

| 条文 | 正确定位 | 理由 |
| --- | --- | --- |
| 第1174条（受害人故意） | 否认因果关系 | 否认"加害行为→损害"的因果链（损害系受害人自己造成） |
| 第1175条（第三人原因） | 否认因果关系 | 否认"加害行为→损害"的因果链（损害系第三人造成） |
| 第181条（正当防卫） | 抗辩不法性阻却 | 引入新事实（对方先有不法侵害）→ 阻却不法性 |
| 第182条（紧急避险） | 抗辩不法性阻却 | 引入新事实（存在现实危险）→ 阻却不法性 |
| 第1173条（与有过失） | 抗辩减责 | 引入新事实（受害人对损害的发生/扩大有过错） |
| 第192条（时效） | 抗辩行使阻止 | 引入新事实（时效期间已届满） |
| "未尽作为义务"的否认 | 否认 | 否认"不作为"要件事实（已尽到义务 = 不存在不作为） |
| "无避免可能性" | 否认因果条件性 | 否认条件关系（即使作为也无法避免损害） |

### 6.3 结构位置规则

- **否认**：置于被否认要件的正下方，缩进一级，前缀"否认："
- **抗辩**：独立成项，与被抗辩层级的其他要件平级或新建子层，前缀"抗辩："
- **反抗辩**：置于抗辩项下方，缩进一级，前缀"反抗辩："

## 七、跨阶段规范双重挂载检查

问题：某些规范在检视程式的不同阶段具有不同功能（如第1173条同时作为成立阶段减责事由和责任范围阶段减损事由）。如果仅挂载一处，将遗漏其部分功能。

### 7.1 操作规则

完成要件拆解后，逐一检查所有已识别的辅助/防御规范：

对每一个已挂载的规范：

**Q**: 该规范的条文文字是否涵盖了检视程式中多个阶段/层级的情形？

- 是 → 在每个相关位置分别挂载，注明各处的具体功能差异
- 否 → 维持单点挂载

### 7.2 常见双重挂载规范

| 规范 | 位置1 | 位置2 | 功能差异 |
| --- | --- | --- | --- |
| 第1173条 | 责任成立阶段 | 责任范围阶段 | 位置1：被侵权人对损害的发生有过错 → 减轻；位置2：对损害的扩大有过错 → 限缩赔偿范围 |
| 第557条第1款 | 合同消灭抗辩 | 侵权消灭抗辩 | 债的通用消灭事由，适用于一切债权 |
| 第192条 | 合同行使抗辩 | 侵权行使抗辩 | 通用时效抗辩（但第196条排除部分物权请求权适用） |

```
  └── 如是：争议焦点为[具体问题]
辅助规范：[列出解释/补充该要件的条文]
举证责任：[原告/被告]
下位子要件：[如有继续拆解]
```
